# kadai05_api
作業工程進捗アプリ

## ②課題内容（どんな作品か）
- 設備・電気工事などの建設現場向けに作成した、作業工程進捗共有アプリです。
職人さんがスマホから作業報告を行うことで、現場の進捗情報をリアルタイムで一元管理できます。

- 現場では、職人さんから口頭・LINE・電話などで共有された進捗を、施工管理者が工程表や出来高表で手作業により管理しているケースが多く、二重入力や情報の分断が発生しています。
このアプリでは、現場のやり方を大きく変えることなく、
職人さん → 作業報告
Firebase → データ保存
GAS → スプレッドシート連携
管理者 → 確認・承認

という流れを構築することで、報告から管理までをスムーズにつなげることを目指しました。

- また、進捗率が25%、50%、75%、100%の節目に達し、担当者確認が完了すると、システムが自動で通知を投稿します。

## ⑤工夫した点・こだわった点
- 現場に馴染みやすいUI設計：
- 建設現場ではITツール導入のハードルが高いため、入力の手間を極力なくせるよう、職人さんがスマホから直感的に操作できるシンプルな入力UIを採用しました。
新しい操作を覚えなくても直感的に入力できるようにしています。

- Firebaseを正本データにした設計：
- 前回はスプレッドシート連携をベースに構築していましたが、データ整合性の問題が発生しました。
今回はFirebase Realtime Databaseを正本データとして設計し、そこからスプレッドシートへ同期する構成に変更しました。

これにより、

・データ消失リスク低減
・リアルタイム反映
・今後のAPI拡張

を目指しました。

- 管理者承認フローの実装：
- 施工管理者や担当者がスプレッドシート上で確認チェックを行う運用にしました。
チェック後はFirebaseへ自動反映され、アプリ側にもリアルタイムで反映されるようにしています。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- 難しかった点：
- 最も苦労したのは、Firebase・GAS・スプレッドシート間のデータ整合性です。
特に、
スプレッドシート列変更
チェックボックス状態保持
Firebaseキー管理
非同期通信
の部分で、データ構造のズレによるバグが発生しました。
一つひとつログを確認しながら原因を切り分けて修正しました。

- 次回トライしたい機能：
・Firebase Authenticationによるログイン管理
・業者別・工種別のアクセス制限
・削除ボタンの実装
・全体進捗率グラフ
・ガントチャート表示
・Weather APIによる天候リスク管理
・写真管理アプリとのAPI連携
・LINE通知機能 

将来的には、単なる進捗共有アプリではなく、設備サブコン向け工程管理OS「MEPla」の中核機能として育てていきたいです。
